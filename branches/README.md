## 分支管理结构

每个分支目录包含以下标准文件：

```
branches/
├── [branch-name]/
│   ├── README.md          # 分支概述和状态
│   ├── todo.md           # 详细任务清单
│   ├── progress.md       # 进展报告 (可选)
│   ├── issues/           # 相关工单 (可选)
│   └── docs/             # 分支文档 (可选)
```

### 文件说明
- **README.md**: 分支的核心说明，包含目标、进展、验收标准等
- **todo.md**: 详细的任务分解和进度跟踪
- **progress.md**: 定期进展报告和里程碑总结
- **issues/**: 分支相关的工单和问题记录
- **docs/**: 分支特定的文档和技术资料

## 分支生命周期

### 1. 创建阶段
- 明确分支目标和验收标准
- 建立初始任务清单
- 设置优先级和时间计划

### 2. 开发阶段
- 每日更新任务进度
- 每周生成进展报告
- 及时识别和解决问题

### 3. 验证阶段
- 执行完整验收测试
- 准备合并申请
- 文档和知识转移

### 4. 合并/终止阶段
- 合并到主分支或终止分支
- 清理相关资源
- 总结经验教训

## 使用指南

### 快速查找分支
```bash
# 查看所有分支状态
cat branches/README.md

# 进入特定分支目录
cd branches/main/
cd branches/dev-env-reform-v1.0/

# 查看分支任务进度
cat branches/main/todo.md
```

### 分支推进流程
1. **选择目标分支**: 根据优先级和状态选择要推进的分支
2. **查看当前状态**: 阅读README.md了解分支概况
3. **检查任务清单**: 查看todo.md了解具体任务
4. **执行推进工作**: 根据任务清单执行相应工作
5. **更新进度**: 完成后更新相关文档

### 状态标识说明
- ⭐ **当前活跃**: 正在重点推进的分支
- 🔄 **实施中**: 正在积极开发的分支
- ⚠️ **探索中**: 实验性分支，需要持续评估
- 💤 **空闲**: 等待依赖条件的分支
