# 工作区文档架构总览

> **协议版本**：工作区上下文协议 V1.5.0（2025-11-29）
> **文档版本**：1.0.0（2025-12-14）
> **维护者**：AI Agent Team

## 概述

本工作区采用结构化文档体系，实现知识积累、问题跟踪、报告管理的标准化与可搜索性。文档架构遵循知识库分类策略，支持模块化管理和版本控制。

## 核心架构

### 目录结构

```
docs/
├── knowledge/           # 知识库（按领域分类）
│   ├── 00_index.md     # 知识库索引
│   ├── system/         # 系统架构知识
│   ├── strategy/       # 策略开发知识
│   ├── backtest/       # 回测分析知识
│   ├── ai_agent/       # AI代理知识
│   ├── deployment/     # 部署运维知识
│   └── misc/           # 其他知识
├── issues/             # 工单文档
├── reports/            # 报告文档
└── templates/          # 文档模板
    ├── knowledge_template.md
    ├── issue_template.md
    └── report_template.md
```

### 知识库分类策略

| 分类 | 说明 | 适用场景 |
|------|------|----------|
| system | 系统架构、基础设施 | 技术栈选型、系统设计 |
| strategy | 策略开发、算法逻辑 | 量化策略、交易模型 |
| backtest | 回测分析、性能评估 | 策略验证、风险分析 |
| ai_agent | AI代理、自动化工具 | 智能助手、工作流 |
| deployment | 部署运维、监控告警 | 生产环境、运维管理 |
| misc | 其他专项知识 | 工具使用、经验总结 |

## 使用指南

### 文档编写规范

- **格式要求**：UTF-8编码，CRLF换行
- **标题层次**：单个H1标题，全文唯一
- **列表缩进**：使用2个空格缩进
- **表格对齐**：保持列对齐，内容简洁
- **文件命名**：使用小写字母、下划线分隔

### 知识库管理

1. **分类存放**：按知识领域存放至对应子目录
2. **索引更新**：新增知识时同步更新 `00_index.md`
3. **版本控制**：重要知识点记录版本变更
4. **关联链接**：相关知识间建立引用链接

### 工单与报告

- **工单文档**：存放于 `issues/` 目录，记录问题分析和解决方案
- **报告文档**：存放于 `reports/` 目录，包含分析报告和总结文档
- **模板使用**：优先使用 `templates/` 中的标准模板

## 维护约定

### 文档质量

- **准确性**：确保技术内容的准确性
- **完整性**：包含必要的上下文和示例
- **时效性**：定期review并更新过时内容
- **可读性**：使用清晰的结构和语言

### 协作规范

- **提交信息**：使用规范的commit message格式
- **分支管理**：feature分支开发，PR review合并
- **冲突解决**：及时处理文档冲突，确保内容一致性
