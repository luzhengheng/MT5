# [AI-EXEC] 初始化新版工作区文档结构（融入知识库分类策略）

> **协议版本**：工作区上下文协议 V1.5.0（2025-11-29）
> **执行入口**：云端中介服务器（Alibaba Cloud Linux 3.2104 LTS 容器优化版） + Cursor Desktop
> **安全要求**：禁止明文密钥；所有路径相对 /root/MT5
> **路径规范**：全部使用相对路径 + 正斜杠
> **写作规范**：严格遵循 WRITING_RULES.md（UTF-8, CRLF, 单 H1, 列表缩进 2 空格, 表格对齐）

---

## 背景（Why）

为统一工作区文档体系，提升知识积累、问题跟踪、报告管理的结构化与可搜索性，采用新版文档结构方案（融入知识库分类策略）。初始化目录结构 + 核心文件，确保后续所有文档生成/迁移有标准路径和模板。

## 范围（Scope）

**纳入**：
- 创建 docs/ 顶级目录及完整子目录结构
- 生成 docs/README.md（总览 + 目录树）
- 生成 docs/knowledge/00_index.md（知识库索引）
- 生成 docs/templates/ 下核心模板（knowledge/issue/report）
- 确保与现有文档兼容

**排除**：
- 不修改/迁移现有文件内容
- 不生成具体知识内容
